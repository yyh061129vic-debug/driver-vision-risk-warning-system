# 仓库结构与职责边界

## 处理链路

系统规划的处理链路为：图像与自车状态接入、质量检查、可行驶区域分割、已知障碍物检测、未知异常发现、距离估计、行驶走廊预测、空间冲突与风险计算、状态与日志输出。

当前提交只建立承载该链路的工程边界，不包含算法实现。

## 源码模块

| 模块 | 职责 |
| --- | --- |
| `data` | 数据索引、读取、转换、质量与划分检查 |
| `models` | 分割、检测、异常与距离模型适配层 |
| `training` | 训练入口、配置快照和实验追踪 |
| `inference` | 在线与回放推理编排 |
| `evaluation` | 算法、风险和系统性能指标 |
| `interfaces` | 图像、自车、障碍物、风险和日志报文 |
| `simulation` | CARLA 连通、场景、采集与真值导出 |
| `risk` | 行驶走廊冲突与可解释风险状态机 |

## 产物边界

- Git 内保存：源码、配置、索引、元数据、权重索引、测试和文档。
- Git 外保存：数据本体、模型权重、训练日志、预测、视频和报告产物。
- 外部产物必须由索引、版本和校验值关联，确保可追溯与可复现。
