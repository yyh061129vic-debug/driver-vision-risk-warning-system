## 一、本周工作概述

本周完成项目工程骨架、候选数据集调研、本地环境基线、道路与异常样例可视化、可行驶区域分割 Demo，以及第一版实验方案冻结。当前已形成“数据登记—样例检查—模型推理—结果校验—实验冻结”的最小可复现链路，并建立组员独立提交目录和 Git 协作约定。

本阶段只证明可行驶区域分割基线能够在本地复现，不代表已完成未知障碍物检测、距离估计、风险状态机、目标硬件实时性验证或车规安全验收。

## 二、本周完成事项

### 1. 建立仓库与标准目录

- 完成 `src/`、`configs/`、`data/`、`metadata/`、`checkpoints/`、`outputs/`、`scripts/`、`tests/` 和 `docs/` 等目录划分。

- 增加仓库结构校验脚本和测试，当前可校验 23 个目录、42 个文件。

### 2. 完成候选数据集调研

- 调研 Cityscapes、BDD100K、Mapillary Vistas、Lost and Found、Fishyscapes、RoadAnomaly21 和 RoadObstacle21 共 7 组候选数据。
- 建立机器可读的数据集登记表，记录规模、标注类型、下载入口、许可限制、建议用途和启用状态。
- 明确 BDD100K 的数据许可仍需单独确认；Cityscapes 和 Lost and Found 仅适用于非商业研究；Fishyscapes、RoadAnomaly21 和 RoadObstacle21 主要用于评估，不作为第一版训练语料。
- 第一阶段样例与评测采用 Lost and Found 和 RoadObstacle21，数据本体不进入 Git。

### 3. 固化本地开发环境基线

- 本地开发机为 Windows 11、Intel Core Ultra 9 185H、31.42 GiB 内存和 RTX 4070 Laptop GPU（8188 MiB）。
- 项目使用 Python 3.12.13、PyTorch 2.13.0+cu130、TorchVision 0.28.0+cu130、Transformers 5.14.0 和 OpenCV 5.0.0.93。
- 已验证 `torch.cuda.is_available()` 为真，SegFormer 可使用 RTX 4070 完成 GPU 推理。
- CUDA Toolkit、ONNX Runtime、TensorRT 和 CARLA Python 包尚未安装。
- 环境采集时 C 盘仅剩 17.67 GiB，后续完整数据集、CARLA 和多份权重应迁移到容量充足的磁盘。

### 4. 完成任务 4：样例可视化

- 本地生成 20 张道路与异常标注叠加图，其中 Lost and Found 10 张、RoadObstacle21 10 张，并生成 1 张总览图。
- 每张图均包含道路区域和异常或障碍物区域；程序校验确认图像可读取、道路像素和障碍像素均非空。
- 固化道路、障碍物和忽略区域的标签映射，生成可版本化样例索引和输入文件 SHA-256 记录。
- 原图及叠加图保留在 Git 忽略目录，遵守 Lost and Found 禁止再分发原图及修改图的限制。

### 5. 完成任务 5：可行驶区域分割 Demo

- 使用固定 revision 的 `nvidia/segformer-b0-finetuned-cityscapes-1024-1024` 模型完成单图和视频入口。
- 单图模式可输出道路掩码、道路边界、置信度图、叠加图和包含版本及哈希信息的 `result.json`。
- 在 RoadObstacle21 的 `validation_34.webp`（1920×1080）上完成真实验收，犬只前景未被划入可行驶区域。
- 本机单样例实测 GPU 延迟为 418.599 ms，CPU 对照为 703.039 ms；该结果仅代表当前样例和本地机器，不作为最终实时性结论。
- 当前默认使用 RTX 4070 GPU，另保留 CPU 配置用于冻结实验对照。

### 6. 冻结第一版实验方案

- 冻结版本为 `phase1_drivable_area_v1 / 1.0.0`。
- 固定模型 revision、权重哈希、512×512 实际输入尺寸、随机种子、标签映射和推理决策规则。
- 固定 40 张本地样例：6 张用于开发冒烟检查，34 张用于留出评测；Lost and Found 按序列隔离，RoadObstacle21 validation 不用于阈值拟合。
- 固定 Binary mIoU、Road IoU 和 Boundary F-score 三项主要质量指标，并规定两个数据域分别报告。
- 需求未给出的质量阈值、延迟预算、风险阈值和目标 GPU 继续保持待确认，不在 V1 中虚构数值门槛。

## 五、当前问题与风险

1. **数据许可风险**：BDD100K 数据条款仍待确认；Lost and Found 与 SegFormer 当前许可证限定非商业研究，不能直接推导为可商业交付。
2. **存储空间风险**：C 盘剩余空间不足以承载完整候选数据集、CARLA 资源和多份权重，需要尽快确定外部存储位置。
3. **数据域偏移风险**：当前 SegFormer 由 Cityscapes 微调，在乡村道路、夜间、极端天气和异常障碍物场景中可能失效。
4. **实时性风险**：RTX 4070 单样例延迟仍不代表目标硬件性能，最终 GPU、延迟预算和部署方式尚未确认。
5. **功能范围风险**：当前仅完成可行驶区域分割；已知障碍物检测、未知风险发现、空间冲突判断和风险预警尚未进入完整联调。
6. **验收口径风险**：质量阈值、风险阈值、摄像头参数和目标场景仍待产品、安全和硬件负责人确认。
