# SegFormer 障碍物类型识别能力调查

- 调查日期：2026-07-21
- 调查对象：本项目当前使用的 `nvidia/segformer-b0-finetuned-cityscapes-1024-1024` 权重及异常检测流水线
- 调查目的：确认模型能够直接命名的障碍物类别、只能作为未知异常发现的类别，以及当前 Demo 的真实输出边界

## 1. 结论摘要

1. 当前 SegFormer 权重共有 19 个 Cityscapes 语义类别，其中可以直接作为交通参与者类别使用的有 8 类：`person`、`rider`、`car`、`truck`、`bus`、`train`、`motorcycle`、`bicycle`。
2. 这 8 类是权重具备的**语义分割标签能力**，不等于当前系统已经完成“障碍物类型识别”。现有 Demo 的 `result.json` 尚未输出这些类别的 `class_id`、`class_name`、实例框或置信度。
3. 动物、纸箱、石块、树桩、遗撒物等没有专用类别。当前能做的是利用能量分数在道路区域内发现“可疑区域”，并统一交给后续系统作为 `unknown` 或 `uncertain`，不能可靠命名具体类型。
4. 墙、围栏、杆、交通标志、植被等虽然有语义类别，但通常是场景组成部分。只有在它们侵入可行驶区域，并经过空间关系或风险规则判断后，才能被视为固定障碍物。
5. 本项目当前 20 张 RoadObstacle21 图片的测试结果说明未知异常检测仍处于实验阶段：共输出 44 个可疑区域，但有 58.597% 的真实障碍物像素仍被分为道路，暂不能宣传为稳定的障碍物识别器。

## 3. 按障碍物类型划分的当前能力

### 3.1 有直接类别标签，可作为“已知类型”候选

- 行人：`person`
- 骑行者：`rider`
- 小汽车：`car`
- 卡车：`truck`
- 公交车：`bus`
- 轨道车辆：`train`
- 摩托车：`motorcycle`
- 自行车：`bicycle`

这里的“可识别”表示模型可以给像素分配上述语义标签。要形成系统接口所需的障碍物对象，还需要从类别图中提取连通区域，并输出类别、边界框、面积和平均置信度。当前代码尚未完成这一步。

### 3.2 没有直接类别标签，只能尝试发现为“未知异常”

| 实际物体示例 | 当前可做的事 | 当前不能保证的事 |
|---|---|---|
| 狗、猫、鹿等动物 | 若与训练分布不同，可能在道路内产生较高异常分数 | 不能输出动物种类，甚至不能保证稳定检出 |
| 纸箱、包裹、行李、遗撒货物 | 可能输出一个未知可疑区域 | 不能区分纸箱、袋子或货物类型 |
| 石块、砖块、混凝土块 | 可能输出一个未知可疑区域 | 不能输出“石块”或材料类别 |
| 树枝、木头、树桩 | 可能输出未知区域，或被误分成植被/道路 | 不能稳定区分倒伏树枝与路旁植被 |
| 雪橇、玩具、异形物体 | 可能利用能量分数发现分布外区域 | 不能命名物体类型 |
| 锥桶、路障、施工设施 | 没有专用标签，可能被异常分支发现 | 不能稳定输出“锥桶”或“路障”类别 |
| 婴儿车、轮椅、电动滑板车 | 可能被近似分成 rider、bicycle 等，或作为异常 | 不能保证细分类别正确 |
| 坑洞、积水、冰雪、油污 | 可能表现为道路外观异常 | 当前没有专门的路面病害类别和验证结果 |

未知异常分支的正确接口含义应是“道路上存在需要进一步处理的可疑区域”，而不是“已经识别出该物体是什么”。

### 3.3 有场景类别，但不能直接等同为障碍物

`wall`、`fence`、`pole`、`traffic light`、`traffic sign`、`vegetation` 可以被模型分割，但这些类别在正常位置通常不是危险物。需要至少同时满足“位于或侵入道路区域”以及后续距离、轨迹或碰撞风险条件，才能进入风险状态机。

## 4. 当前 Demo 实际输出能力

当前单图结果的异常区域包含：

- `bbox_xyxy`：可疑区域边界框；
- `area_pixels`：区域面积；
- `mean_anomaly_score`：平均异常分数。

当前没有输出：

- `class_id`；
- `class_name`；
- 已知障碍物实例列表；
- 动物、纸箱、石块等未知物体的细分类别；
- 跟踪 ID、距离、速度或碰撞时间。

因此，目前对外准确表述应为：

> 权重具备 19 类城市场景语义分割能力，其中 8 类为交通参与者；当前项目已实现道路分割和道路内未知异常区域提取，尚未把交通参与者类别转换成障碍物类型接口，也不能为未知物体命名。

## 5. 数据集能验证什么

本地数据集信息来自 `metadata/dataset_registry.yaml`。

| 数据集 | 现有标签范围 | 能验证 | 不能验证 |
|---|---|---|---|
| Lost and Found | 障碍物/自由空间像素标签 | 遗撒类小障碍物定位、道路占用 | 纸箱、石头等细分类别识别 |
| RoadObstacle21 | obstacle / not-obstacle / void 和道路 ROI | 道路内异常或障碍物定位 | 具体物体类型分类 |
| RoadAnomaly21 | anomaly / not-anomaly / void | 场景中异常区域定位 | 动物、车辆、物体的细分类别；当前本地尚未启用 |

这些数据集主要提供二值异常/障碍物标签，不能用来计算各类障碍物的分类准确率。若要证明“能识别纸箱、狗、石块等类型”，需要另建带细分类别标注的测试集。

## 6. 最近 20 张图的证据

结果文件：`outputs/task5_segmentation_anomaly/batch_extra_20_filtered_heatmap/summary.json`。

- 样本数：20；
- 异常评分：Energy；
- 道路掩码腐蚀：5 像素；
- 类别边界抑制：8 像素；
- 输出可疑区域总数：44；
- 道路 IoU：99.688%；
- 障碍物排除率：41.403%；
- 障碍物被预测为道路的比例：58.597%。

“44 个区域”不等于“识别了 44 个障碍物”，也不包含障碍物类型。该结果说明道路区域分割较稳定，但未知障碍物的排除和异常区域提取仍需继续调参与验证。

## 7. 建议的下一步接口

在不改变 V1 冻结对照配置的前提下，可以将系统拆成两条输出分支：

1. **已知障碍物分支**：从完整 `class_map` 中提取 ID 11–18 的连通区域，输出 `class_id`、`class_name`、`bbox_xyxy`、`area_pixels` 和 `mean_confidence`。
2. **未知障碍物分支**：继续使用道路内 Energy/MSP 异常分数，输出 `obstacle_type: unknown`、`bbox_xyxy`、面积和平均异常分数。
3. **融合与风险判断**：结合道路相交关系、目标距离、连续帧跟踪和风险阈值，生成 `known`、`unknown` 或 `uncertain` 状态。

`configs/system.yaml` 中的 `pending_decisions.known_obstacle_classes` 当前仍为 `null`，所以正式纳入哪些类别应由项目组确认后再冻结，不能在调查阶段直接当作已确定业务规则。

## 8. 最终能力清单

- **模型权重可直接区分的交通参与者：8 类。**
- **当前 Demo 已输出的障碍物类型：0 类。**
- **当前 Demo 可输出的未知风险形式：无类别的道路内可疑区域。**
- **当前数据集可量化验证的目标：障碍物/异常区域定位，不是细分类别识别。**
